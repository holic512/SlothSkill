# 数据库设计报告骨架

适用：SQL Server、数据库课程设计、概念设计/逻辑设计/物理设计为主体的项目。

默认输出倾向：

- 用户需求
- 业务流程分析
- 概念模式设计
- 逻辑模式设计
- 数据库保护设计

章节证据来源分配：

- 题目驱动：用户需求、业务流程、数据管理目标、应用场景
- 证据驱动：实体属性、联系、ER 图、关系模式、约束与完整性设计

关键提醒：

- 用户需求应围绕业务题目和数据管理目标展开
- 不要把现有 SQL 脚本的表名清单直接替换成需求分析

优先覆盖证据：

- 实体、属性、联系
- 业务流程
- ER 图
- 关系模式转换
- 表结构、主键、外键、约束
- 规范化与完整性设计

常用图：

- 业务流程图
- 数据流图
- ER 图
- 表间关系图

输出时避免：

- 把应用层业务逻辑作为主要篇幅
- 在没有 SQL 或实体证据时臆造详细表结构
- 把需求分析写成建表语句说明

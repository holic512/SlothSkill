# Linux 项目报告骨架

适用：Linux 操作系统实验、环境部署、服务管理、Shell 自动化、系统运维类项目。

默认输出倾向：

- 项目背景与目标
- 环境与工具
- 方案设计
- 操作过程与关键命令说明
- 结果展示
- 测试验证与问题处理
- 总结

章节证据来源分配：

- 题目驱动：背景、目标、应用场景、运维需求
- 证据驱动：环境配置、操作过程、关键命令、验证结果

优先覆盖证据：

- Shell 脚本
- 服务部署
- 配置文件
- 用户权限与文件系统操作
- 网络与进程管理
- 运行结果截图或占位

常用图：

- 部署架构图
- 操作流程图
- 服务交互图

输出时避免：

- 堆砌 Linux 基础理论而忽略项目目录证据

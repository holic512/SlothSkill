# Python 实验报告骨架

适用：Python 综合开发、GUI、网络编程、多线程、文件处理、数据库应用类项目。

默认输出倾向：

- 摘要
- 需求分析
- 总体设计
- 详细设计
- 具体实现
- 测试与验证
- 总结

章节证据来源分配：

- 题目驱动：项目背景、目标、需求分析、用户场景、功能需求
- 代码驱动：总体设计、详细设计、具体实现、测试验证

关键提醒：

- 需求分析应从题目、业务问题和使用场景展开
- 不要把 GUI、线程、网络等已有模块直接当作需求分析正文

优先覆盖证据：

- GUI 交互
- 网络/API 调用
- 多线程或异步处理
- 文件读写与持久化
- 数据库与模型映射
- 时序与交互流程

常用图：

- 用例图
- 模块依赖图
- ER 图
- 时序图
- 活动图

输出时避免：

- 用不存在的并发机制强行拔高
- 把通用框架能力写成项目已实现功能
- 把需求分析写成代码结构解说

# Java 实验报告骨架

适用：Java 课程设计、面向对象课程设计、带数据库或 GUI 的 Java 项目分析报告。

默认输出倾向：

- 需求分析
- 系统设计
- 数据定义或数据库设计
- 系统编码及运行
- 系统测试
- 总结

章节证据来源分配：

- 题目驱动：开发背景、意义、目标、需求分析、用户需求、应用场景
- 代码驱动：系统设计、数据库设计、编码实现、运行界面、测试分析

关键提醒：

- 需求分析重点是围绕课题目标、业务问题和用户场景展开
- 不要把现有 Java 模块或方法列表直接改写成需求分析

优先覆盖证据：

- 技术栈与分层结构
- 核心功能模块
- 实体与表结构
- 核心流程与模块交互
- 运行界面截图或占位
- 测试环境、测试用例、问题分析

常用图：

- 功能结构图
- 用例图
- 组件图
- 活动图
- 时序图
- ER 图

输出时避免：

- 直接粘贴大段源码
- 把通用 Java 知识写成项目事实
- 把需求分析写成模块功能解释

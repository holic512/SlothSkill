# 软件项目管理报告骨架

适用：招标、合同、WBS、进度、成本、质量、风险、配置管理、执行控制类文档。

默认输出倾向：

- 项目开发背景
- 项目招标
- 项目开发合同
- 需求规格
- WBS 任务分解
- 成本计划
- 进度计划
- 质量计划
- 配置管理计划
- 风险计划
- 集成计划
- 执行控制

章节证据来源分配：

- 题目驱动：项目背景、目标、需求规格、招标与合同背景
- 证据驱动：WBS、进度、成本、质量、风险、配置与执行控制资料

优先覆盖证据：

- 项目基本信息
- 角色与职责
- 计划类文档
- 流程与里程碑
- 风险与控制措施

常用图：

- 流程图
- 甘特逻辑图
- 组织结构图
- 风险矩阵图

输出时避免：

- 在无项目信息时虚构甲乙方和预算细节

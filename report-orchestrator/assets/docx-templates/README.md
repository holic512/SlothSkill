# DOCX Templates

这个目录用于存放 `pandoc --reference-doc` 使用的 Word 模板。

维护规则很简单：

- 用户没有给出明确样式要求时，默认使用 `0-general.docx`
- 用户给出的排版要求与某个模板的样式特征接近时，优先匹配该模板
- 新增模板时，只需要补模板文件、`manifest.json` 条目和本 README 的模板清单

## 当前模板

| 编号 | 文件名 | 适用场景 | 样式摘要 |
|---|---|---|---|
| 0 | `0-general.docx` | 默认通用模板；用户未指定排版要求时使用 | 使用模板内置 Normal、Heading、目录样式，作为兜底参考模板 |
| 1 | `1-songti-times-heading.docx` | 课程设计、实验报告、规范化书面报告 | 一级标题黑体小二，二级标题黑体小三，三级标题黑体四号，正文宋体小四，行距 1 |

## 匹配规则

- 如果用户显式指定模板编号或模板名，优先使用指定模板
- 如果用户给出样式要求，按 `manifest.json` 中的 `match_keywords` 选择最接近模板
- 如果没有足够的样式信息，使用 `0-general.docx`
- 如果多个模板同时命中，优先选择命中关键词更多的模板；仍有冲突时回退到 `0-general.docx`

## 新增模板流程

1. 把新的 `.docx` 模板放进本目录
2. 按 `编号-slug.docx` 形式命名
3. 在 `manifest.json` 中增加一条记录
4. 在本 README 的“当前模板”表中补一行说明
5. 如新增模板的匹配条件较特殊，再同步更新 skill 的模板选择说明

## 样式维护注意事项

- `reference.docx` 依赖 Word 样式，不需要模板里放完整报告正文
- 至少保证 `Normal`、`Heading 1/2/3`、目录、表格等关键样式稳定
- 模板中只保留少量示例元素即可，避免把无关正文内容带入导出结果
- Office 生成的 `.~*.docx` 临时文件不要纳入模板资产

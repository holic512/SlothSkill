# Generic Experiment Template

Use this template when the repository still looks like an implementation-backed experiment project, but the language or structure is mixed.

Suggested structure:

- Project overview
- Requirements and objectives
- Repository and technology analysis
- Module and data design
- Implementation overview
- Runtime or usage notes
- Testing or validation plan
- Conclusion

Guidance:

- stay conservative
- prefer supported detail over coverage
- do not force language-specific structure when the evidence is mixed

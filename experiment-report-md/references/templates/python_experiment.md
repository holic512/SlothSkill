# Python Experiment Template

Use this template for Python coursework or lab projects when Python is clearly the primary implementation language.

Suggested structure:

- Background and objectives
- Requirements analysis
- Overall design
- Module design
- Data or persistence design when present
- Implementation details
- Testing and validation
- Conclusion

Evidence to prioritize:

- package layout and entry scripts
- dependency manifest and runtime environment
- API, GUI, threading, async, file I/O, or database behavior
- tests, logs, or reproducible execution steps

Common diagrams:

- module dependency diagram
- sequence diagram
- activity diagram
- ER diagram when persistence is central
